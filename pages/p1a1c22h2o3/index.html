# 网络基础

网络请求过程

  1. IP地址

    整个网络传输比作快递，数据就是包裹

    ip地址就是门牌号

  2. 动态ip和静态ip

    ip地址格式：xxx.xxx.xxx.xxx，总长32位 ipv4

    方便理解，每八位做10进制转换

    (局域网ip)192.168.xxx都是私有的 出了局域网就连接不进来

    使用192.168.xxx对外联网，都会公用对外共享的公共ip (公网ip) 进行连接。

    动态ip是自动分配的，静态ip一般是你去买阿里云服务器的ip

    所以：一般服务器不会随便封ip地址 (例，学校有人爬淘宝网，淘宝是不会直接把学校的封了。

    但可以限制

  3. url协议

    格式： 协议//域名:端口/访问资源路径？key=参数过滤&key=参数2#锚点(同一页面)

    常用协议：http、https、file、ftp

    域名：ip地址的别名，可直接填写ip地址

    相对url和绝对url：

    ​ 相对url是在html里面做跳转的url 路径 :

    ​ / 开头 默认会自动添加上域名,否则会在当前的url的后面加上这个

    ​ 绝对url：就是完整url

  4. 七层网络协议：应用层=>表示层=>会话层=>传输层=>网络层=>数据链路层=>物理层

    常用的是五层协议：应用=>传输(包装)=>网络(路由)=>数据链路层(交换机)=>物理层(双绞线、集线器)

    发送:左到右=> 接收：右到左：<=

    应用层：http、ftp、pop3、DNS

    传输层：TCP、UDP

    网络层：ICMP、IP、IGMP

    数据链路层：ARP、RARP

    物理层：物理传输介质

    如何编码使用?

    集线器、路由器、操作系统继续实现了协议，只需调用接口即可

    接口: socket编程

    其中还有更复杂的接口，如果一般是底层开发才需要接触到(路由器开发等..)

  5. TCP/IP协议：

    ip地址是逻辑地址，可以修改变化，代表不了bobby的不变地址。

    bobby是网卡的map地址(从出生就给厂家设置好的)

    如果里主的人换了，是管不了的

    Tcp协议：传输控制协议，把数据分解成一定长度的”片段“,发给IP地址。

    tcp是一种面向连接的单播协议，发送数据前需要建立连接（客户端和服务器的内存需要保存一份关于对方的信息，例ip、端口等）tcp可看成一种字节流，处理IP层或以下层，为了防止丢包、重复、错误等问题，在建立链接时，需要交换链接参数，参数可放在TCP头部。

    IP协议：用于将多个数据包连接起来，对所在的源地址和目的地址之间传输数据包 (路由)

    相当快递员为传输过来的”片段“贴上标签(源地址和目的地址)，然后发往中转站...

    路由器：连接互联网和各局域网的设备，对根据信道情况自主选择设置路由（最佳路径）

    每个节点上的路由器会记录与自己相邻的路由器，形成一个路由表（动态数据库，网络活动中不断更新）

    网络中是：广播协议。当数据包到达一个路由器，路由器会获取这个数据包的信息，以及查询路由表，然后选择合适路径进行发送数据包。

    总结TCP；提供可靠、面向连接、字节流、传输层的服务，采用三次握手建立一个连接、采用4此挥手来关闭一个连接

  6. socket编程：插座的意思

    插座中的火线、地线、中线可以理解为ip地址/端口等。

    socket本身不是一个协议，是一套接字，操作系统为了方便大家直接使用tcp协议而创建的抽象层，

    把复杂tcp/ip协议隐藏在接口后面

    具体逻辑：

    socket实际就是在内存中记录下俩边的ip和端口等信息。实际上不是真正的连接。

  7. http协议：超文本传输协议：应用层协议

    是万维网协会和internet工作小组LETF合作的结果，最终发布一些列的RFC。

    都是基于TCP协议上进行开发定制的比我们下面自定义协议复杂的多

    所以我们要基于http协议要遵守他提供的规则。

    是一个无状态的协议，单向的

    特点：

    • 简单快速：只需要传入请求方法和路径。
    • 灵活：http可以传递任意类型的数据对象。通过content-type指定。
    • 无连接：每处理一个球球，服务器会自动断开连接，节省传输时间和服务器压力。
    • 无状态：协议对事务处理没有记忆能力，需要通过cookie和session来加以区别。
    • 支持B/S和C/S模式

    具体格式：

    浏览器已经帮我们解析好了

    在控制台的network也可以查看相应的信息(可以进行信息的过滤)

    如果点击 request(请求) Headers 的view source 就可以查看跟上面的格式了

    get方法：参数包含在url中，回退时无害，只能进行url编码，参数有长度限制，不安全

    post方法：参数通过request body传递，回退会再次提交请求，支持多种编码格式，没有长度限制

    content-type: 常见三种模式

    • application/x-www-from-urlencoded:常见的post提交数据方式 ：默认
    • nultipart/from-data post提交数据：from的enctype等于这个值 ：文件
    • application/json json格式

#-*- codeing = utf-8 -*-
#qq服务器
"""
功能:
1. 转发消息
2. 处理登录
3. 处理退出
4. 维护历史消息，维护在线用户维护用户连接
"""
import socket
from collections import defaultdict #和dic用法一样 也可以传入list,方法
import threading
import json
#维护用户连接
online_users=defaultdict(dict)#如果key不存在会生成一个key 防止报错
#维护用户的历史消息
user_msgs=defaultdict(list)
server=socket.socket()
#绑定ip
server.bind(("0.0.0.0",8000))
server.listen()

#处理连接
def bandle_sock(sock,addr):
    while True:
        data=sock.recv(1024)
        json_data=json.loads(data.decode("utf8"))
        action=json_data.get("action","")
        if action=="login":#如果是登录
            online_users[json_data["user"]]=sock #把用户的连接放入在线字典中
            sock.send("登录成功！".encode("utf8"))
        elif action == "list_user":
            #获取当前在线用户
            all_users=[user for user,sock in online_users.items()]
            #变成字符串
            sock.send(json.dumps(all_users).encode("utf8"))
        elif action=="history_msg":
            sock.send(json.dumps(user_msgs.get(json_data["user"],[])).encode("utf8"))
            # if json_data["user"] in action:
            #     sock.send(json.dumps(user_msgs[json_data["user"]]).encode("utf8"))
        elif action=="send_msg":
            if json_data["to"] in  online_users:
                online_users[json_data["to"]].send(json.dumps(json_data).encode("utf8"))
            user_msgs[json_data["to"]].append(json_data)
        elif action=="exit":
            del online_users[json_data["user"]]
            sock.send("退出成功！".encode("utf8"))
#接收多用户的连接
while True:
    # 堵塞
    sock,addr=server.accept()
    #启动一个线程去处理新用户连接
    client_thread=threading.Thread(target=bandle_sock,args=(sock,addr))
    client_thread.start()#启动线程

因为http协议是无状态的协议，识别每个用户需要用cookie和session用来保证。

cookie和seesion：

用户登录成功后给用户返回一段字符串,且在服务器or内存维护用户session id和这个字符串。

cookie在http协议的一段消息，写在http的headers中，里面有id和name，和sessionid，以及过去时间，交给浏览器维护

下次请求会将cookie的参数全部带上

seesion和cookie的区别：

  • seesion：有服务器维护，服务器解释，通过set-cookie交给浏览器
  • cookie：是浏览器的工具，重set-cookie保存到本地，并在后续每次请求中都带上这些参数

# Scrapy

是基于基于Twisted的异步处理框架，Twisted是异步网络IO库，所以性能高。

python内置异步IO库是asyncio

基本使用

1.创建scrapy项目：
	终端输入 scrapy startproject 项目名称
	
2.项目组成：
	spiders
		__init__.py
		自定义的爬虫文件.py->由自己创建，是实现爬虫核心功能的文件
	__init__.py
    items.py ‐>  定义数据结构的地方，继承自scrapy.Item的类 		middlewares.py -> 中间件 代理
    pipelines.py -> 管道文件 只有一个类，用于处理下载数据的后续处理 默认是300优先级，值越小优先级越高（1‐1000）
    settings.py -> 配置文件，例：是否遵守robots协议，User‐Agent定义等
    
3.创建爬虫文件：
	（1）跳转到spiders文件夹 cd 目录名字/目录名字/spiders
	（2）scrapy genspider 爬虫名字 网页的域名
  爬虫文件的基本组成：
  		继承scrapy.Spider类
  			name = 'baidu' ‐‐‐》 运行爬虫文件时使用的名字
  			allowed_domains ‐‐‐》 爬虫允许的域名，在爬取的时候，如果不是此域名之下的 url，会被过滤掉
  			start_urls ‐‐‐》 声明了爬虫的起始地址，可以写多个url，一般是一个
  			parse(self, response) ‐‐‐》解析数据的回调函数
  				response.text ‐‐‐》响应的是字符串
  				response.body ‐‐‐》响应的是二进制文件 
  				response.xpath()‐》xpath方法的返回值类型是selector列表 
  				extract() ‐‐‐》提取的是selector对象的是data 
  				extract_first() ‐‐‐》提取的是selector列表中的第一个数据
 
4. 运行爬虫文件：
	scrapy crawl 爬虫名称 
	注意：应在spiders文件夹内执行

架构组成：

（1）引擎 ‐‐‐》自动运行，无需关注，会自动组织所有的请求对象，分发给下载器
（2）下载器 ‐‐‐》从引擎处获取到请求对象后，请求数据 
（3）spiders ‐‐‐》Spider类定义了如何爬取某个(或某些)网站。
			包括了爬取的动作(例 如:是否跟进链接)以及如何从网页的内容中提取结构化数据(爬取item)。 
			换句话说，Spider就是您定义爬取的动作及 分析某个网页(或者是有些网页)的地方。
（4）调度器 ‐‐‐》有自己的调度规则，无需关注 
（5）管道（Item pipeline） ‐‐‐》最终处理数据的管道，
		会预留接口供我们处理数据 当Item在Spider中被收集之后，它将会被传递到Item Pipeline，
		一些组件会按照一定的顺序执行对Item的处理。 每个item pipeline组件
		(有时称之为“Item Pipeline”)是实现了简单方法的Python类。他们接收到Item并通过它执行 一些行为，
		同时也决定此Item是否继续通过pipeline，或是被丢弃而不再进行处理。 
		以下是item pipeline的一些典型应用：
        	1. 清理HTML数据
            2. 验证爬取的数据(检查item包含某些字段) 
            3. 查重(并丢弃) 
            4. 将爬取结果保存到数据库中
