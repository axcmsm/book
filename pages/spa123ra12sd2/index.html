# sparkcore

RDD详解：

弹性分布式数据集、spark中的基本抽象、表示不可变、

在内部每个rdd都有五个特征：

  • 分区列表
  • 用于计算每个分割的函数
  • 对其他rdd的依赖关系列表
  • 键值RDD的分区器，（例如：说RDD是哈希分区的）
  • 计算拆分的首选位置列表（例如：HDFS文件）

RDD创建：

  • 方式：

    1. 并行化本地集合
2. 引用加载外部存储系统（hdfs、hive、hbase、kafka、es)数据集

sc.textFile(本地/hdfs/文件夹，分区数)//不要读取大量小文件
sc.wholeTextFiles(本地/hdfs文件夹,分区数)//专门读取小文件

RDD操作

  • transformation：转换生成新的rdd。不会立即执行，需要action触发
  • action：无返回值，所有的action函数立即执行。
基本操作：
map、faltmap、filter、foreach、saveAsTextFile

分区操作：
mapPartitions(每个分区，一个分区开启一次) 代替map(针对分区内每条一条数据开启一次)
foreachPartitions 代替foreache(针对分区内每条)

重分区操作：
增加、减少分区：repartion (底层调用coalesce(numpartitions,shuffle)) 会产生shuffle
减少分区：coalesce (默认只能减少,不会产生shuffle)#添加true 页可以改成增加

聚合操作：
	没有key聚合：sum、reduce、fold、aggregate
	有key聚合：groupbykey+sum/reduce、reduceBykey(有预聚合)、flodByKey、aggregateBykey

关联函数：pairRDDFunctions
	xxxjoinxxx()：内连接、外连接(左右)、全连接等

排序：
	sortBy (第二个参数true为正序,).take(前几个)
	sortBykey 
	top

持久化/缓存：热点数据  (本地，相对较快，保留rdd依赖关系)
	rdd.cache //=>rdd.persist
	rdd.persist //=>下面
	rdd.persist(StorageLevel.MEMORY_ONLY)//persist底层默认是缓存
	rdd.persist(StorageLevel.MEMORY_AND_DISK)//内存+磁盘(内存不够才用)
	rdd.unpersist()//清空缓存
	
检查点：放在hdfs上面 （可靠，相对较慢，不保留依赖关系）
	sc.setCheckPointDir(hdfs路径)//设置检查点路径
	rdd.checkpoint()//对频繁的rdd进行缓存

>>开发中持久化+检查点可联合使用：先持久化+再进行检查点缓存
	
共享变量:
	广播变量：变量在所有节点的内存之间进行共享，在每个机器上缓存一个只读副本、而不是机器上的每个任务都生成一个副本
	val data=sc.broadcast(广播变量)
	data.value #获取广播数据
	累加器：累加器支持所有节点之间进行累加计算
	val count=sc.longAccumulator(名称) #计数器
	count.add(1) #累加

外部数据源：	JDBC
	文本文件、对象文件、sequence文件、文件系统、数据库

spark内核原理:

依赖关系：

spark的核心是根据RDD来实现的，spark scheduler是spark核心实现的重要一环，其作用是任务调度。

根据rdd的依赖关系(容错)构建DAG、基于DAG划分Stage，每个Stage中的任务发到指定节点运行。

宽依赖:有shuffle,（父rdd的一个分区会被子Rdd的多个分区所依赖）

对其进行state阶段划分，同一个stage阶段中都是窄依赖，可对窄依赖进行优化。

窄依赖:没shuffle,（父rdd的一个分区会被子Rdd的1个分区所依赖)

可进行优化：合并操作、形成管道，同一个管道中各个操作可以由一个线程执行完，提高容错。

DAG:spark任务/程序的执行流程、从新创建rdd开始、到Action结束。(有几个Action、就有几个Dag)

Stage:把DAG划分成相互依赖的多个Stage、划分依据是RDD之间的宽依赖。state由一组并行的task组成。一个task处理一串分区数据

名称介绍：

application:应用(编写代码)
driver：驱动程序
sparkcontext：spark上下文环境，用来和clusterManager进行通信，并进行资源申请，任务分配和监控等。
clusterManage：集群管理器
worker：工作节点
executor：运行worker的jvm进程
rdd:弹性分布式数据集
dag:有向无环图：根据action形成rdd的执行流程图
job:作业,按照Dag进行就形成了job--按照图动态的执行
stage:Dag中，根据shuffle以拉划分出来的一个个执行阶段
task:一个分区上的一系列操作(管道流水线操作)
taskset：任务集，同一个stage中的各个task组成的集合。

案例代码：

package sparkCore
import com.hankcs.hanlp.HanLP
import org.apache.spark.{SparkConf, SparkContext}
object SOGO {
  def main(args: Array[String]): Unit = {
    val conf = new SparkConf().setAppName("sougou").setMaster("local[*]")
    val context = new SparkContext(conf)
    context.setLogLevel("error")
    val lines = context.textFile("data/input/sogo.txt")
    //todo 切割每一行
    val data = lines.map(item => {//map 输入一行输出一行
      val arr = item.split("\\s+")
      Sougouy(arr(0), arr(1), arr(2), arr(3).toInt, arr(4).toInt, arr(5))
    })
    //todo 1.分割分词
    val wordsRdd = data.flatMap(record => { //输入一行输出多行
      val str = record.queryWords.replaceAll("\\[|\\]", "")
      import scala.collection.JavaConverters._
      HanLP.segment(str).asScala.map(_.word) //分词
    })
    //统计分词
    val countWordsBysortTop10 = wordsRdd
      .filter(word=> !word.equals(".") && !word.equals("+"))
      .map((_, 1)).reduceByKey(_ + _)
      .sortBy(_._2, false)
      .take(10)//前10

    //todo 2.用户热门搜搜词
    val hotbyUserRDD = data.flatMap(record => { //输入一行输出多行
      val str = record.queryWords.replaceAll("\\[|\\]", "")
      import scala.collection.JavaConverters._
      val words = HanLP.segment(str).asScala.map(_.word)//同一条数据
      val userid = record.userId
      words.map(item=>(userid,item))
    })
    //统计
    val hotbyUserBytop10 = hotbyUserRDD.filter(t => !t._2.equals(".") && !t._2.equals("+"))
      .map((_, 1)).reduceByKey(_ + _).sortBy(_._2, false).take(10)

    //todo 3.按照时间统计
    val timeSortBytop10 = data.map(item => { //统计到小时即可
      (item.queryTime.substring(0, 5), 1)
    }).reduceByKey(_ + _).sortBy(_._2, false).take(10)

    //打印
    countWordsBysortTop10.foreach(println)
    println("-----------------------------")
    hotbyUserBytop10.foreach(println)
    println("-----------------------------")
    timeSortBytop10.foreach(println)
    context.stop()//关闭
  }
  case class Sougouy(queryTime:String,userId:String,queryWords:String,resultBank:Int,clickRank:Int,clickUrl:String)
}

# sparkStreaming

概述：

计算模式

  • 原生流处理：所有输入的记录会一条接着一条被处理，storm和flink都是采用这种方式

    （flink优化:设置缓存块大小阀值毫秒级触发数据块传输，可以平衡延迟可高吞吐量，做到低延迟，高吞吐）

  • 微批处理：将输入的数据一某个时间间隔T，切割成多个微批量数据，然后对批量数据进行处理：spark采用（秒级)

数据抽象:DStream=>seq[RDD]

DStream表示一个连续的数据流。DStream由一系列连续的rdd表示，每个rdd都包含特定的间隔数据。

API:

  • transformation

    大多数和rdd中的类似、map、flatmap、filter、count、reduce、conuntByvalue、reduceBykey、union、join、coroup

  • ouput/action

    xxxsavexxx

object WordCount3 {
  def createFunc():StreamingContext={
    //准备环境
    val conf: SparkConf = new SparkConf().setAppName("spark").setMaster("local[*]")
    val sc = new SparkContext(conf)
    sc.setLogLevel("warn")
    val context = new StreamingContext(sc, Seconds(5))//每五秒分批一次

    //持久化累加的状态 检查点
    context.checkpoint("./ckp")


    //加载数据
    val lines = context.socketTextStream("master", 9999)

    //处理数据

    //自定义累加函数
    val updateFunc=(currentValues:Seq[Int],historyValue:Option[Int])=>{
      if(currentValues.size>0){
        val currentResult =  currentValues.sum+historyValue.getOrElse(0)//取不出来赋值为0
        Some(currentResult)
      }else{
        historyValue
      }
    }


    val count = lines.flatMap(_.split(" ")).map((_, 1))
      .updateStateByKey(updateFunc)


    //todo 状态管理
    //输出结果
    count.print()
    context
  }

  def main(args: Array[String]): Unit = {
    //todo 在检查点加载数据如果加载不到调用创建方法
    val context = StreamingContext.getOrCreate("./ckp", createFunc _)
    context.sparkContext.setLogLevel("ERROR")


    //启动并等待结束
    context.start()//todo 启动
    context.awaitTermination()//开启等待接收数据
    //关闭资源
    context.stop(stopSparkContext = true,stopGracefully = true)//优雅关闭（把一批都处理掉再关)
  }
}

  • 窗口函数

    窗口长度：计算多久

    滑动间隔：每隔多久计算一次

    窗口长度10s > 滑动间隔5s：每隔5s计算最近10秒的数据--滑动窗口

    窗口长度10s > 滑动间隔10s: 每隔10s计算最近10秒的数据--滚动窗口

    窗口长度10s< 滑动间隔15s：每隔15s计算最近10秒的数据--会丢失数据，不用

    //处理数据

      top3.foreach(println)
      println("==============top3======")
      value1
    })

    //输出结果

    //关闭资源

  • 方式2：direct approach（直接连接kafka分区获取数据，大大提高并行力，调用低阶kafka的api，offset自己存储和维护，消除了zk不一致情况）

api：kafka0.10.x版本只支持Direct模式（0.8淘汰）

object SparkStreamingKafka {
  def main(args: Array[String]): Unit = {
    //准备环境
    val conf: SparkConf = new SparkConf().setAppName("spark").setMaster("local[*]")
    val sc = new SparkContext(conf)
    sc.setLogLevel("warn")
    val context = new StreamingContext(sc, Seconds(5))//每五秒分批一次

    //加载数据-从kafka
    val kafkaParams = Map[String, Object](
      "bootstrap.servers" -> "master:9092",
      "key.deserializer" -> classOf[StringDeserializer],
      "value.deserializer" -> classOf[StringDeserializer],
      "group.id" -> "sparkkafka",//消费者组名称
      "auto.offset.reset" -> "latest",//偏移量重置，如果有offset记录、从offset开始消费
//      "auto.commit.interval.ms"->"1000",//自动提交间隔偏移量
      "enable.auto.commit" -> (false: lang.Boolean)//释放自动提交
    )
    val topic = Array("first")//订阅主题

    //todo 消费kafka数据
    val kafkaDs = KafkaUtils.createDirectStream[String, String](
      context,
      LocationStrategies.PreferConsistent, //位置策略
      ConsumerStrategies.Subscribe[String, String](topic, kafkaParams) //消费策略
    )

    //处理数据
    //todo 手动提交偏移量
    kafkaDs.foreachRDD(rdd=>{
      if(!rdd.isEmpty()){
        //消费
        rdd.foreach(record=>{
          val topic = record.topic()
          val partition = record.partition()
          val offset = record.offset()
          val key = record.key()
          val value = record.value()
          val info = s"""topic:${topic},partition:${partition},offset:${offset},key:${key},value:${value}"""
          println("消费到的详细信息数据:"+info)
        })
        //todo 提交偏移量
        val offsetRanges = rdd.asInstanceOf[HasOffsetRanges].offsetRanges//获取偏移量
        kafkaDs.asInstanceOf[CanCommitOffsets].commitAsync(offsetRanges)//todo 手动提交偏移量到默认主题

        //可以保存到mysql

        println("当前批次数据已消费,并手动提交默认主题")
      }
    })

//      val value = record.value()
//      val info = s"""topic:${topic},partition:${partition},offset:${offset},key:${key},value:${value}"""
//      info
//    }
//    )

    //输出结果
//    infos.print()
    //启动并等待结束
    context.start()//todo 启动

    //关闭资源

# 性能调优

文档：https://spark.apache.org/docs/latest/tuning.html (opens new window)

九项基本原则：

  • 避免创建重复的RDD
  • 尽可能复用同一个RDD
  • 对多次使用的RDD进行持久化
  • 尽量避免使用shuffle类算子（例：reduceBykey、join、distinct、repartition等）、join解决办法：对小数据进行广播
  • 使用map-side预聚合的shuffle操作（使用reduceBykey、不使用groupbykey）
  • 提高算子性能（reduceBykey代替groupbykey、mapPartitions代替map、foreachpartitions代替foreach、使用filter后再进行coalesce）
  • 广播中等规模的变量
  • 使用kryo序列化提高性能（spark2.0后简单的类型已经默认使用了)
  • 优化数据结构（尽量用字符串代替对象、使用原始的类型(int、long)代替字符串）

N多配置参数：

  • 提交参数

    num-executors ：每个spark作业一般设置50~100个左右
executor-memory : 内存4g到8g合适，实际情况看具体环境
executor-cores :2到4个合适，实际情况看具体环境(num-executors*executor-cores)不要操作队列总cpu的1/3~1/2
driver-memory :默认1g
