spark-Streaming笔记

# 简介

数据处理的方式：

  • Streaming流式数据处理
  • 批量(batch)数据处理

数据处理延迟的长短：

  • 实时数据处理：毫秒级别
  • 离线数据处理： 小时/天

💨⛳SparkStreaming 是准实时（秒,分钟），微批次（时间）的数据处理框架👩🏾👮🏾‍♀️

使用离散化流抽象表示,就是在来到Dirver之前封装了一层采集器DStream（离散化流）

DStream是随时间推移而收到的数据的序列，其内部每个时间区间收到的数据都作为 RDD 存在

所以被称为：离散化

特点：

  • 可支持输入数据多：Kafka、flume、Twitter、zeroMQ和简单的套接字等等
  • 可使用spark的高度抽象原语(方法)：map、reduce、window、jon等进行运算
  • 可保存多地方：HDFS、数据库等
  • 易用，容错、易整合

架构

  • 整体架构

  • Streaming 架构图

背压机制：防止接收数据和消费数据不均衡，可动态调节Receiver数据的接受率

spark.streaming.backpressure.enabled 控制是否启用 backpressure 机制

默认值false，即不启用

自定义数据源

用法：需要继承 Receiver，并实现 onStart、onStop 方法来自定义数据源采集。

def main(args: Array[String]): Unit = {
    //1.初始化 Spark 配置信息
    val conf = new SparkConf().setMaster("local[*]").setAppName("RDDStream")
    //2.初始化 SparkStreamingContext
    val ssc = new StreamingContext(conf, Seconds(4))
	//创建	
    val message: ReceiverInputDStream[String] = ssc.receiverStream(new MyReceiver())
    message.print()//打印
    ssc.start()
    ssc.awaitTermination()
  }

  /**
   * 自定义数据采集器
   */
  class MyReceiver extends Receiver[String](StorageLevel.MEMORY_ONLY){
    private var float=true
      //最初启动的时候，调用该方法，作用为：读数据并将数据发送给 Spark
    override def onStart(): Unit = {
      //启动一个独立线程
      new Thread(new Runnable {
        override def run(): Unit = {
          while (float){
            val  message="采集是数据:"+new Random().nextInt(10).toString
            store(message)//存储
            Thread.sleep(500)
          }
        }
      }).start()
    }

    override def onStop(): Unit = {
        float=false;
    }
  }

# 转化

DStream分为Transformations（转换）和 Output Operations（输出）两种

操作与 RDD 的类似

无状态 : 只能对单个区段进行累加操作

  • map
  • flatmap
  • filter
  • repartition
  • reduce.....

有状态：可记录上一个时间段的计算结果再次累加操作

  • UpdateStateByKey:记录历史记录
 //todo 创建环境对象
    val conf = new SparkConf().setMaster("local[*]").setAppName("streaming")
    val ssc=new StreamingContext(conf,Seconds(10))
    // todo 第二个是批量处理周期
    val datas= ssc.socketTextStream("localhost", 9999)
    ssc.checkpoint("cp")//缓冲区目录 使用有状态需要设定检查点路径
    val worToone = datas.map((_, 1))
//    val wordtocount = worToone.reduceByKey(_ + _)//无状态
    //updateStateByKey根据数据的状态进行更新
    //参数一相同key的value数据
    //第二个参数变送表示缓冲区相同的key的value
   val state=worToone updateStateByKey ((seq: Seq[Int], buff:Option[Int]) =>{
      val newCount=buff.getOrElse(0)+seq.sum
      Option(newCount)//放入缓冲区
    })
  state.print()
    //驱动采集器
    ssc.start()
    //等待采集器关闭
    ssc.awaitTermination()

其他方法

  • transform：可以将底层rdd获取后进行转换操作

    在Dirver端执行，且会周期性执行。 采集一次执行一次

  • join：俩个不同的流连接到一起 其实就是俩个rdd的join

  • windowOperations：把多个采集周期当成一个整体，依次滑动，默认步长为一个周期

    滑动整个整体,数据由少到多,再变少

    window(时间，步长(时间)) 时间范围是采集周期的整数倍，Seconds(6)

    当步长大于等于这个时间就没有重复数据了

  • Window:

    （1）window基于对源 DStream 窗化的批次进行计算返回一个新的 Dstream；
（2）countByWindow: 返回一个滑动窗口计数流中的元素个数；
（3）reduceByWindow: 通过使用自定义函数整合滑动区间流元素来创建一个新的单元素流；
（4）reduceByKeyAndWindow(func, windowLength, slideInterval, [numTasks]): 当在一个(K,V)
对的 DStream 上调用此函数，会返回一个新(K,V)对的 DStream，此处通过对滑动窗口中批次数
据使用 reduce 函数来整合每个 key 的 value 值。
（5）reduceByKeyAndWindow(func, invFunc, windowLength, slideInterval, [numTasks]): 这个函
数是上述函数的变化版本，每个窗口的 reduce 值都是通过用前一个窗的 reduce 值来递增计算。

# 数据输出

没有输出操作会提示错误

  • print ：打印
  • savexxx ：保存
  • foreachRdd：拿到最底层的rdd进行操作 不会出现时间戳

注意

  • 连接不能写在 driver 层面（序列化）
  • 如果写在 foreach 则每个 RDD 中的每一条数据都创建，得不偿失
  • 增加 foreachPartition，在分区创建（获取）

# 关闭

当程序逻辑发生改变时需要停止程序。

优雅关闭：对象.stop(true,true) 第二个参数当计算节点计算完现有的数据再关闭

需要在启动时，另外启动一个新的线程，来关闭采集器，跟主线程独立开，

否则会影响正常采集，new Thread(new Runable(代码))

这个线程里面还要加入判断读取第三方存储的数据来决定是否需要关闭

例如：hdfs、redis、mysql、zk...等待 来决定采集器是否关闭

除此之外，还要获取当前sparkstreaming的状态再判断

ssc.getStarts() 如果在线关闭，否则就不会了，获取不到睡眠一下

线程也要停一下 system.exit(0)

恢复数据：

getActiveOrCreate("cp",()=>[//检查点
代码  数据保存到检查点中
])
ssc.checkpoint("cp")//恢复数据

    package com.axcmsm.streaming.dome.utils
import java.sql.{Connection, PreparedStatement, ResultSet}
import java.util.Properties
import javax.sql.DataSource
import com.alibaba.druid.pool.DruidDataSourceFactory


object JdbcUtil {
  //初始化连接池
  var dataSource: DataSource = init()
  //初始化连接池方法
  def init(): DataSource = {
    val properties = new Properties()
    properties.setProperty("driverClassName", "com.mysql.jdbc.Driver")
    properties.setProperty("url", "jdbc:mysql://hadoop111:3306/spark-streaming")
    properties.setProperty("username", "root")
    properties.setProperty("password","密码")
    properties.setProperty("maxActive",
     "50")
    DruidDataSourceFactory.createDataSource(properties)
  }
  //获取 MySQL 连接
  def getConnection: Connection = {
    dataSource.getConnection
  }
  //执行 SQL 语句,单条数据插入
  def executeUpdate(connection: Connection, sql: String, params: Array[Any]): Int
  = {
    var rtn = 0
    var pstmt: PreparedStatement = null
    try {
      connection.setAutoCommit(false)
      pstmt = connection.prepareStatement(sql)
      if (params != null && params.length > 0) {
        for (i <- params.indices) {
          pstmt.setObject(i + 1, params(i))
        }
      }
      rtn = pstmt.executeUpdate()
      connection.commit()
      pstmt.close()
    } catch {
      case e: Exception => e.printStackTrace()
    }
    rtn
  }
  //执行 SQL 语句,批量数据插入
  def executeBatchUpdate(connection: Connection, sql: String, paramsList:
  Iterable[Array[Any]]): Array[Int] = {
    var rtn: Array[Int] = null
    var pstmt: PreparedStatement = null
    try {
      connection.setAutoCommit(false)
      pstmt = connection.prepareStatement(sql)
      for (params <- paramsList) {
        if (params != null && params.length > 0) {
          for (i <- params.indices) {
            pstmt.setObject(i + 1, params(i))
          }
          pstmt.addBatch()
        }
      }
      rtn = pstmt.executeBatch()
      connection.commit()
      pstmt.close()
    } catch {
      case e: Exception => e.printStackTrace()
    }
    rtn
  }
  //判断一条数据是否存在
  def isExist(connection: Connection, sql: String, params: Array[Any]): Boolean =
  {
    var flag: Boolean = false
    var pstmt: PreparedStatement = null
    try {
      pstmt = connection.prepareStatement(sql)
      for (i <- params.indices) {
        pstmt.setObject(i + 1, params(i))
      }
      flag = pstmt.executeQuery().next()
      pstmt.close()
    } catch {
      case e: Exception => e.printStackTrace()
    }
    flag
  }
  //获取 MySQL 的一条数据
  def getDataFromMysql(connection: Connection, sql: String, params: Array[Any]):
  Long = {
    var result: Long = 0L
    var pstmt: PreparedStatement = null
    try {
      pstmt = connection.prepareStatement(sql)
      for (i <- params.indices) {
        pstmt.setObject(i + 1, params(i))
      }
      val resultSet: ResultSet = pstmt.executeQuery()
      while (resultSet.next()) {
        result = resultSet.getLong(1)
      }
      resultSet.close()
      pstmt.close()
    } catch {
      case e: Exception => e.printStackTrace()
    }
    result
  }
  //主方法,用于测试上述方法
  def main(args: Array[String]): Unit = {
  } }

实现实时的动态黑名单机制：将每天对某个广告点击超过 100 次的用户拉黑。
注：黑名单保存到 MySQL 中。
思路分析
1）读取 Kafka 数据之后，并对 MySQL 中存储的黑名单数据做校验；

3）在存入 MySQL 之后对数据做校验，如果单日超过 100 次则将该用户加入黑名单

    package com.axcmsm.streaming.dome

import com.axcmsm.streaming.dome.utils.JdbcUtil
import org.apache.kafka.clients.consumer.{ConsumerConfig, ConsumerRecord}

import java.util.Date

/**
 *
 * 需求一：广告黑名单
 * 实现实时的动态黑名单机制：将每天对某个广告点击超过 100 次的用户拉黑。
 *  注：黑名单保存到 MySQL 中。

object Streaming_req_blacklist {
  def main(args: Array[String]): Unit = {
    //1.初始化 Spark 配置信息
    val conf = new SparkConf().setMaster("local[*]").setAppName("RDDStream")
    //2.初始化 SparkStreamingContext
    val ssc = new StreamingContext(conf, Seconds(4))
    //3.定义 Kafka 参数
    val kafkaPara: Map[String, Object] = Map[String, Object](
      ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG ->
        "hadoop111:9092,hadoop112:9092,hadoop112:9092",
      ConsumerConfig.GROUP_ID_CONFIG -> "axcmsm",
      "key.deserializer" ->
        "org.apache.kafka.common.serialization.StringDeserializer",
      "value.deserializer" ->
        "org.apache.kafka.common.serialization.StringDeserializer"
    )

    val kafkaData: InputDStream[ConsumerRecord[String, String]] = KafkaUtils.createDirectStream[String, String](
      ssc,
      LocationStrategies.PreferConsistent,
      ConsumerStrategies.Subscribe[String, String](Set("axcmsm"), kafkaPara)
    )
    //获取数据了
    val adClickdata = kafkaData.map(
      kafkadata => {
        val data = kafkadata.value()
        val datas = data.split(" ")
        AdClickData(datas(0), datas(1), datas(2), datas(3), datas(4))
      }
    )

    // todo 1.判断黑名单 周期性
    // todo 判断点击是否在黑名单中
    // todo 如果不在 添加到统计数量表
    // todo 再对统计数据量的数据进行判断 是否拉入黑名单
    val ds = adClickdata.transform(
      rdd => {
        //周期性获取黑名单数据
        val back_list = ListBuffer[String]()
        val conn = JdbcUtil.getConnection
        //查询字段
        val ps: PreparedStatement = conn.prepareStatement("select userid from black_list")
        val rs = ps.executeQuery() //查询
        while (rs.next()) {
          back_list.append(rs.getString(1))
        }
        rs.close()
        ps.close()
        conn.close()
        //过滤
        val filterRdd = rdd.filter(
          data => {
            //TODO 判断是否在里面  如果不在保留
            !back_list.contains(data)
          }
        )
        //TODO 统计
        val count = filterRdd.map( //转化
          data => {
            //因为是毫秒值 需要转化
            val sdf = new SimpleDateFormat("yyyy-MM-dd")
            val day = sdf.format(new Date(data.ts.toLong))
            val user = data.user
            val ad = data.ad
            ((day, user, ad), 1)
          }
        ).reduceByKey(_ + _) //聚合
        count
      }
    )


    ds.foreachRDD(rdd=>{


      //一个分区创建一个连接对象 优化
//  rdd.foreachPartition(
//    iter=>{
//      val conn=JdbcUtil.getConnection
//      iter.foreach{
//        case (((day, user, ad), count))=>{
//
//        }
//      }
//      conn.close()
//    }
//  )
      rdd.foreach{
        case (((day, user, ad), count))=>{
          println(s"${day} ${user} ${ad} ${count}")
          if(count>=30){
              //拉人黑名单
              val conn = JdbcUtil.getConnection
            val sql="""
                      |insert into black_list (userid) values(?)
                      |ON DUPLICATE KEY
                      |UPDATE userid=?
                      |""".stripMargin
              JdbcUtil.executeUpdate(conn,sql,Array(user , user))
              conn.close()
          }else{
            //查询数据库有更新
            val conn = JdbcUtil.getConnection
            val sql="""
                  |select *
                  |from user_ad_count
                  |where dt=? and userid =? and adid =?
                  |
                  |""".stripMargin
            val flg=JdbcUtil.isExist(conn,sql,Array(day,user,ad))
            if (flg){
              val sql2 ="""UPDATE user_ad_count SET count=count+? WHERE dt=? AND userid =? AND adid =? """
              JdbcUtil.executeUpdate(conn,sql2,Array(count,day,user,ad))

              //判断更新后数据没有没有超过
              val sql3="""
                         |select * from user_ad_count
                         |where dt=? and userid=? and adid=?
                         |and count>=30
                         |""".stripMargin
              val flg2 = JdbcUtil.isExist(conn, sql3, Array(day, user, ad))
              if (flg2){
                //拉人黑名单
                val sql4="""
                          |insert into black_list (userid) values(?)
                          |ON DUPLICATE KEY
                          |UPDATE userid=?
                          |""".stripMargin
                JdbcUtil.executeUpdate(conn,sql4,Array(user , user))
              }

            }else{
              //没有插入
              val sql5="""
                         |insert into user_ad_count (dt,userid,adid,count)
                         |values(?,?,?,?)
                         |""".stripMargin
              JdbcUtil.executeUpdate(conn,sql5,Array(day , user,ad,count))
            }
            conn.close()
          }
        }
      }
    })
    ssc.start()
    ssc.awaitTermination()
  }
