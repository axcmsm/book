# 🦏DDL

数据定义语言(主要掌握建表使用)：CREATE、ALTER与DROP 操作对象包括数据库、表、视图....

建表完整语法树：

解释：

（1）CREATE TABLE 创建一个指定名字的表。如果相同名字的表已经存在，则抛出异常；
用户可以用 IF NOT EXISTS 选项来忽略这个异常。
（2）EXTERNAL 关键字可以让用户创建一个外部表，在建表的同时可以指定一个指向实
际数据的路径（LOCATION），在删除表的时候，内部表的元数据和数据会被一起删除，而外
部表只删除元数据，不删除数据。
（3）COMMENT：为表和列添加注释。
（4）PARTITIONED BY创建分区表
（5）CLUSTERED BY创建分桶表
（6）SORTED BY不常用，对桶中的一个或多个列另外排序
（7）ROW FORMAT DELIMITED  (默认的分隔符是\001)
	[FIELDS TERMINATED BY char] 字段之间的分隔符 
	[COLLECTION ITEMS TERMINATED BY char] 集合之间的分隔符
	[MAP KEYS TERMINATED BY char] （map映射kv之间分隔符）
	[LINES TERMINATED BY char]     行数据之间的分隔符
	| SERDE serde_name 
	[WITH SERDEPROPERTIES (property_name=property_value, property_name=property_value, ...)]
	用户在建表的时候可以自定义SerDe或者使用自带的SerDe。如果没有指定ROW FORMAT 或者ROW FORMAT DELIMITED，将会使用自带的SerDe。在建表的时候，用户还需要为表指定列，用户在指定表的列的同时也会指定自定义的SerDe，Hive通过SerDe确定表的具体的列的数据。
SerDe是Serialize/Deserilize的简称，hive使用Serde进行行对象的序列与反序列化。
（8）STORED AS指定存储文件类型
常用的存储文件类型：SEQUENCEFILE（二进制序列文件）、TEXTFILE（文本）、RCFILE（列式存储格式文件）
如果文件数据是纯文本，可以使用STOREDASTEXTFILE。如果数据需要压缩，使用STORED AS SEQUENCEFILE。
（9）LOCATION ：指定表在HDFS上的存储位置。
（10）AS：后跟查询语句，根据查询结果创建表。
（11）LIKE允许用户复制现有的表结构，但是不复制数据。

外部表：hive只管理元数据，不会管理真正的数据，删除表即删除元数据 关键字external (数据不易用外部)

内外部表转换：

  1. 查询表类型 desc formatted student2;
  2. 修改内部表 student2 为外部表 alter table student2 set tblproperties('EXTERNAL'='TRUE');
  3. 修改外部表 student2 为内部表 alter table student2 set tblproperties('EXTERNAL'='FALSE');

  • 加载数据方式

    • 创建分桶表

    • 对数据进行高效抽样

其他语法：

  • 数据库DDl操作

    • create database 数据库名 （默认路径**/user/hive/warehouse/*.db**)

    • desc database 显示数据信息

    • use database 切换数据库

    • drop database 删除数据库 [cascade|restrict]

      [CASCADE|RESTRICT]表示是否需要强制删除该库及其相关的表，默认为RESTRICT。

    • alter database 修改Hive中的数据库关联的元数据

  • 表DDL操作

    • desc table 显示建表语句 （formattion,以表格格式显示元数据)
    • drop table 删除表
    • truncate table 删除所有行保留数据结构
    • alter table 修改表结构

  • 分区DDL操作(Alter table)

    • add partition 更改表元数据 (需要保证增加的分区位置路径下，数据已经存在，或者增加完分区之后导入分区数据)

    • rename partition 分区重命名

    • drop partition 删除分区

    • alter partition 修改分区

    • MSCK partition 检查分区元数据修复

允许使用复杂的联接表达式,支持非等值连接、同一查询中可以连接2个以上的表

如果每个表在联接子句中使用相同的列，则Hive将多个表上的联接转换为单个MR作业

join时的最后一个表会通过reducer流式传输，并在其中缓冲之前的其他表，因此，将大表放置在最后有助于减少reducer阶段缓存数据所需要的内存

在join的时候，可以通过语法STREAMTABLE提示指定要流式传输的表。如果省略STREAMTABLE提示，则Hive将流式传输最右边的表 SELECT STREAMTABLE(a) a.val, b.val, c.val FROM a JOIN b ON (a.key = b.key1) JOIN c ON (c.key = b.key1)

join在WHERE条件之前进行。

SELECT /*+ MAPJOIN(b) */ a.key, a.value FROM a JOIN b ON a.key = b.key
