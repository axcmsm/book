Oozie笔记

# 简介

Oozie驯象人,工作流调度软件,需要自己部署,xml配置

官网:https://oozie.apache.org/ (opens new window)

模块：三个配置文件

  • workflow:顺序流程节点、支持fork(分开)、join(合并) (核心)
  • coordinator:定时触发workflow
  • Bundle job: 绑定多个coordinator

workflow常用节点:

  • control node 控制流节点：start、end、kill、fork、join、decision等

  • action node 动作节点 具体任务类型：mr spark shell java hive shell等

结合起来描绘出一个工作流 DAG图

# 部署

  • 配置httos服务
  • 配置jobhistory服务
  • 重启hadoop相关服务
  • 解压oozie安装包
  • 分发

因为版本不兼容问题,可以用cdh版本进行安装，此处不做介绍

# 基本使用

解压官方指定的oozie-example.tar.gz包

进入examples里面的apps有很多配置模板(需要用到修改即可)

  • 调用shell脚本

    1.进入`examples`里面的`apps`目录包shell目录拷贝到你指定的配置目录
2.里面有俩个文件:job.properties和workflow.xml
 job.properties 配置nd、yn、放置脚本的目录
 workflow.xml 配置任务相关信息 俩个文件之间可以用${}调用定义的变量
 执行文件也需要用${调用}
3.把这三个文件一同上传到你指定的目录下
4.执行任务
bin/oozie job -oozie 发布任务地址 -config job.properties路径  -run
5.web页面查看任务
6.杀死任务
bin/oozie job -oozie 发布任务地址 -kill 任务id
