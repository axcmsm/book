# azkaban简介

官网https://azkaban.github.io/ (opens new window)

用作调度hadoop的工作流程

常见工作流调度系统：Crontab(简单任务)、Ooize、azkaban、airflow、dolphinscheduler等

oozie：oozie使用xml配置,在cdh中也可以使用hue组件进行可视化操作
azkaban：yaml配置
airflow：python脚本
dolphinscheduler:可视化

使用步骤：描述工作流程、配置定时规则

Oozie和azkaban对比：

oozie是一个重量级任务调度系统、功能全面、配置复杂

azkaban是一个轻量级调度器

# 部署

架构:web server(调度、用户、权限管理等)+Executor server(具体任务执行)+mysql(定时规则、任务状态等)

部署：单机、集群

集群规划

hadoop111  hadoop112   hadoop113
  web       Executor    Executor

脚本：

#!/bin/bash
# 启动exec集群脚本 里面设置了web关闭 没有设置web开启因为web开启之前需要激活
case $1 in
"start")
		for host in master slave1 slave2;
		do
			echo "-------$host start azkaban-------"
			ssh $host "cd /opt/module/azkaban/azkaban-exec ; bin/start-exec.sh"		
		done
		
		echo "正在等待激活....."

		sleep 8
		
		for host in master slave1 slave2;
		do
 			echo "-------$host curl azkaban-------"
			curl -G "$host:12321/executor?action=activate" && echo
		done

		echo "start web "
		ssh slave2  "cd /opt/module/azkaban/azkaban-web ; sh  bin/start-web.sh "
;;
"stop")
		
		echo "stop web"

		ssh slave2  "cd /opt/module/azkaban/azkaban-web ; bin/shutdown-web.sh"

		for host in master slave1 slave2;
		do
			echo "-------$host stop azkaban-------"
			ssh $host "cd /opt/module/azkaban/azkaban-exec ; bin/shutdown-exec.sh"
		done
;;
esac

# 条件工作流

按照自定义条件执行job任务

条件构成：

  • (多个)父任务运行结束输出的参数
  • 使用预定义宏(使用它提前预定的内置规则)

基本原理:

（1）父 Job 将参数写入 JOB_OUTPUT_PROP_FILE 环境变量所指向的文件 json字符串 
（2）子 Job 使用 ${jobName:param}来获取父 Job 输出的参数并定义执行条件

再使用条件运算符进行必对执行.

2
3
4

步骤：

  • 编写父任务脚本 json字符串k,v > $JOB_OUTPUT_PROP_FILE

  • 子任务脚本

  • 配置文件

    #子任务添加
condition: ${父任务名称:在文件中的key} == 值   
 #这里是去JOB_OUTPUT_PROP_FILE根号key拿到value 在进行条件判断

# 注意事项

在集群模式下,多台exec执行任务时,是webserver选择其中一个exec去执行。(大概率执行失败)

因为你执行的脚本不在同一个exec节点下。

解决方案：

  • 指定特定exec去执行 （id不固定,负载均衡失效...）

    在mysql的`azkaban数据库的executors表中,查询hadoop111的exec的id
在web页面执行工作流程时添加useExecutor的属性 把id放里面
#id每次启动又概率会发送改变
    1
    2
    3

  • 所有的exec节点都添加该脚本 (推荐)
