    # DockerFile

    DockerFile：用来构建docker镜像的文件，命令参数脚本、相当于原代码

    dockerimages：是通过DockerFile构建生成的镜像。

    docker容器：是镜像运行起来提供的服务器

    基本介绍：

    • 每个关键字(指令)都必须大写字母
    • 执行从上到下顺序执行
    • #表示注释
    • 每个指令都会创建提交一个新的镜像层、并提交

    构建步骤：

    1. 编写dockerfile文件
    2. docker build 构建称为一个镜像
    3. docker run 运行镜像
    4. docker push 发布镜像

    指令:

    FROM #基础镜像
MAINTAINER #镜像谁写的：姓名+邮箱
RUN #镜像构建时需要运行的命令
ADD #添加内容 例:添加tomcat
WORKDIR #镜像的工作目录
VOLUME #挂载的容器卷位置
EXPOSE #暴露端口配置
CMD  #指定这个容器启动的时候要运行的命令，只有最后一个会生效，可被替代
ENTRYPOINT #指定这个容器启动的时候要运行的命令，可以追加命令
ONBUILD #当构建一个被继承 DockerFile 这时会被运行ONBUILD的指令，触发指令
COPY #类似ADD命令，将文件内容拷贝到镜像中
ENV #构建时设置环境变量

    # Docker 网络

    我们安装了docker，就会有一个网卡docker0、桥接模式、采用evth-pair技术，每启动一个docker容器，docker会给docker容器分配一个ip。

    evth-pair：一对虚拟设备接口、成对出现、一端连着协议、一端彼此相连。通常用作一个桥梁。连接各种虚拟网络设备

    结论：容器和容器之间可以相互ping通。

    指定容器名ping通：不需要指定ip

    --link 目标容器名 (反向不能ping通)

    原理：在hosts中添加了映射

    不建议使用

    自定义网络：容器互联 docker network --help

    • 网络模式

      bridge：桥接 docker
none：不配置网络
host：和宿主机共享网络
container：容器网络连通（用的少、局限很大）
      1
      2

      4

    • 命令

      #启动命令会默认添加 --net bridge  默认桥接到docker0   #16可以255*255 24可以255
创建网络
	docker network create --driver bridge --subnet 192.168.0.0/16 --gateway 192.168.0.1 mynet 
	#--driver 模式 
	#--subnet 分配ip范围
	#--gateway 网关  
查看网络
	docker network ls
查看详情
	docker network inspect mynet

使用 例：(可以相互ping通)
	docker run -d -P -name tomcat01 --net mynet tomcat
	docker run -d -P -name tomcat02 --net mynet tomcat
	docker exec -it tomcat01 ping tomcat02

    练习：部署redis集群（分片+高可用+负载均衡）

    #创建redis网段
docker network create redis --subnet 172.30.0.0/16

#通过脚本创建6个redis配置
for port in $(seq 1 6);\
do \
mkdir -p /mydata/redis/node-${port}/conf
touch /mydata/redis/node-${port}/conf/redis.conf
cat << EDF > /mydata/redis/node-${port}/conf/redis.conf
port 6379
bind 0.0.0.0
cluster-enabled yes
cluster-config-file nodes.conf
cluster-node-timeout 5000
cluster-announce-ip 172.38.0.1${port}
cluster-announce-port 6379
cluster-announce-bus-port 16379
appendonly yes
EOF
done
#启动 分别启动
	#1

    -v /mydata/redis/node-1/data:/data \
    -v /mydata/redis/node-1/conf/redis.conf:/etc/redis/redis.conf
    -d --net redis ip 172.30,0.11 redis:5.0.9-alpine3.11 redis-server /etc/redis/redis.conf

    # Docker制作镜像

    dockerfile:

    FROM
MAINTAINER 作者
ENV 设置环境变量 (可以写多条)
RUN 编译镜像时运行的脚本(可多)
CMD 设置容器的启动命令
ENTRYPOINT 设置容器的入口程序
ADD  将宿主机的文件复制到容器内，如果是一个压缩文件，将会在复制后自动解压
COPY 和ADD相似，但是如果有压缩文件并不能解压
WORKDIR 设置工作目录
ARG 设置编译镜像时加入的参数
VOLUMN 设置容器的挂载卷

--- 
RUN：用于指定build 过程中要运行的命令，即是创建 Docker 镜像（image）的步骤
CMD：设置容器的启动命令，只能有一条 CMD 命令，多条则最后一条生效，不支持接收docker run的参数。
ENTRYPOINT：入口程序是容器启动时执行的程序， docker run 中最后的命令将作为参数传递给入口程序 ，ENTRYPOINY类似于 CMD 指令，但可以接收docker run的参数 。
