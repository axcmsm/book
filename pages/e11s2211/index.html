# Elasticsearch概述

es是一个开源高扩展的分布式全文搜索引擎。

官网：https://www.elastic.co/cn/ (opens new window)

采用result风格，传输的json字符串

数据格式：与mysql对比

  • index：数据库
  • documents文档：Row行
  • Fields：字段：column列

倒排索引：根据关键字找到对应id

# linux

  • 单机：上传解压改名

    1. 创建es用户 因为不能用root直接运行（安全问题）
useradd es #新增 es 用户
passwd es #为 es 用户设置密
userdel -r es #如果错了，可以删除再加
chown -R es:es /opt/module/es #文件夹所有者

2. 修改配置文件 
/opt/module/es/config/elasticsearch.yml
    # 加入如下配置
    cluster.name: elasticsearch
    node.name: node-1
    network.host: 0.0.0.0
    http.port: 9200
    cluster.initial_master_nodes: ["node-1"]

修改/etc/security/limits.conf
	# 在文件末尾中增加下面内容
    # 每个进程可以打开的文件数的限制
    es soft nofile 65536
    es hard nofile 65536
    
修改/etc/security/limits.d/20-nproc.conf
    # 在文件末尾中增加下面内容
    # 每个进程可以打开的文件数的限制
    es soft nofile 65536
    es hard nofile 65536
    # 操作系统级别对每个用户创建的进程数的限制
    * hard nproc 4096
    # 注：* 带表 Linux 所有用户名称
    
修改/etc/sysctl.conf
	# 在文件中增加下面内容
    # 一个进程可以拥有的 VMA(虚拟内存区域)的数量,默认值为 65536
    vm.max_map_count=655360	
    
重新加载  sysctl -p

# 进阶

  • 索引:提高搜索性能 倒排索引
  • 文档：一条数据
  • 字段：field
  • 映射：mapping
  • 分片：shards ：一个lucene索引在es里称作分片，一个es索引是分片的集合。es发送查询到每一个lucene索引，然后合并每个分片的结构得到一个全局的结果集。
  • 副本：replicas
  • 分配：allocation

# 🎟

集群：通过elasticsearch-head插件查看集群情况

路由计算：hash(id) % 主分片数量=【0，1，2】

分片控制：用户可以访问任何一个节点获取数据，这个节点称为协调节点

  1. 客户端请求集群节点（任意）-协调节点
  2. 协调节点将请求转换到指定节点
  3. 主分片保存数据
  4. 主分片需要将数据发送个副本
  5. 副本保存后，进行反馈
  6. 主分片进行反馈。
  7. 客户端获取反馈

调节：

  • 客户端发送查询到协调节点
  • 协调节点计算数据所在分片以及全部副本的位置
  • 将请求转发给具体节点
  • 节点返回查询结果，将结果反馈给客户端。

延时：主分片延时+并行写入副本的最大延时

# 优化

Elasticsearch 的基础是 Lucene，所有的索引和文档数据是存储在本地的磁盘中.可在件./config/elasticsearch.yml 中配置。

es是重度使用磁盘的，如果磁盘的吞吐量越大，节点就越稳定。

  • 优化磁盘：SSD、RAID 0、多快磁盘
  • 分片策略：设置合理分片数(节点数<=主分片数*（副本数+1）)、推迟分片分配、
  • 路由选择：带routing查询根据id
  • 写入速度优化：加大translog flush、增加Index Refresh间隔、调整bulk线程池和队列、优化节点分布，优化lucene层的索引建立。
  • 批量数据提交：使用 Bulk 来进行批量写入。
  • 优化存储设备
  • 合理使用合并
  • 减少Refresh次数
  • 加大Flush设置
  • 减少副本数量
  • 内存设置

重要配置

参数名			 参数值 							说明
cluster.name  elasticsearch 配置ES的集群名称，默认值是 ES，会自动发现在同一网段下的集群名称相同的节点
node.name   node-1           集群中的节点名，在同一个集群中不能重复。
node.master true             指定该节点是否有资格被选举成为 Master 节点，默认是 True
node.data   true               指定该节点是否存储索引数据，默认为 True。
index.number_of_shards 1     设置都索引分片个数，默认是 1 片。
index.number_of_replicas  1   设置默认的索引副本个数，默认为 1 个。
transport.tcp.compress true 设置在节点间传输数据时是否压缩，默认为 False，
discovery.zen.minimum_master_nodes 1 设置在选举 Master 节点时需要参与的最少的候选主节点数，默认为 1
discovery.zen.ping.timeout 3s  设置在集群中自动发现其他节点时 Ping 连接的超时时间，默认为 3 秒
