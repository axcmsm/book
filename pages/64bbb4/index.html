Canal入门

# Canal

概述：通过解析mysql的日志来实时同步采集mysql变化的数据

binglog:记录了所有DDL和dml除查询语句的日志、且是事务安全型的。

  • 模式

    statement(语句级):节省空间、可能操作数据不一致
row(行级）:binlog 会记录每次操作后每行的记录的记录、占用较大空间、保证数据的一致性
mixed（statement升级版） :节省空间+兼顾了一定的一致性
    1
    2
    3

    做数据分析级别只能采用row级别，因为无法解析statement的语句级别

# 工作原理

mysql主从复制过程：

  1. master主库将改变的记录、写入二进制日志binarylog中。
  2. slave从库向mysql master 发送的dump协议、将master主库的binary log events 拷贝到它的中继日志relay log。
  3. slave从库中读取并重做了日志中的事件，将改变的数据同步到自己的数据库。

canal工作原理：将自己伪装称一个slave、假装从master复制数据

使用场景：

  1. 异地数据库之间的同步
  2. 更新缓存(读写分离、将缓存数据同步到redis等) row
  3. 抓取业务表中新增变化数据、用作实时报表。 row

  • java代码

    /**
 * 测试canal tcp
 */
public class CanalDemo1 {
    public static void main(String[] args) throws InterruptedException, InvalidProtocolBufferException {
        //todo 获取连接
        CanalConnector canalConnector = CanalConnectors.newSingleConnector(new InetSocketAddress("master", 11111), "example", "", "");

        while (true){
            //获取连接
            canalConnector.connect();
            //监控的该库的所有表
            canalConnector.subscribe("demo1.*");
            //获取数据 一次拉取多少条数据
            Message message = canalConnector.get(100);

            //获取entry
            List<CanalEntry.Entry> entries = message.getEntries();
            if (entries.size()<=0){
                System.out.println("没有抓取到数据等待5秒");
                Thread.sleep(5000);
            }else {
                //解析遍历数据
                for (CanalEntry.Entry entry : entries) {
                    //表名
                    String tableName = entry.getHeader().getTableName();
                    //类型
                    CanalEntry.EntryType type = entry.getEntryType();
                    //序列化后的数据
                    ByteString storeValue = entry.getStoreValue();
                    //判断数据是否为row
                    if (CanalEntry.EntryType.ROWDATA.equals(type)){
                        //5. 反序列化数据
                        CanalEntry.RowChange rowChange = CanalEntry.RowChange.parseFrom(storeValue);
                        //6. 获取当前事件的操作类型
                        CanalEntry.EventType eventType = rowChange.getEventType();
                        //7.获取数据集r
                        List<CanalEntry.RowData> rowDatasList = rowChange.getRowDatasList();
                        //8. 遍历并打印数据集合
                        rowDatasList.stream().forEach(x->{
                            JSONObject beforData = new JSONObject();//更新前
                            List<CanalEntry.Column> beforeColumnsList = x.getBeforeColumnsList();
                            beforeColumnsList.stream().forEach(column -> {
                                beforData.put(column.getName(),column.getValue());
                            });

                            JSONObject afterData = new JSONObject();//更新后
                            List<CanalEntry.Column> afterColumnsList = x.getAfterColumnsList();
                            afterColumnsList.stream().forEach(column -> {
                                afterData.put(column.getName(),column.getValue());
                            });
                            //打印
                            System.out.println("table"+tableName+",EventType:"+eventType
                                            +",beforData:"+beforData+",afterData:"+afterData );
                            //也可用写入kafka
                        });
                    }else{
                        System.out.println("当前操作类型"+type);
                    }
                }
            }
        }
    }
}
