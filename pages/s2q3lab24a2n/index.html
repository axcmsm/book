# sparksql简介

用于处理结构化数据处理的spark模块

提供了俩个编程抽象

分布式数据集：

  • DataFrane：类似二维表格,懒执行，DataFrane管理的结构(元数据)，rdd是管理数据内容
  • DataSet：对DataFrane做了优化，添加了类型
  • 例：DS(对象)=>DF(属性)=>RDD(属性里面值)

DataSet其实里面包含了DataFrane,不知道以后会不会被丢弃。

区别：

  • RDD : 一般和mllib同时使用,且不支持sparksql操作
  • DataFrame：类型固定为一行，df和ds不与mllib同时使用，支持sql，和特殊保存（csv..）
  • DataSet:是DataFrame升级

特点：易整合,统一数据访问、兼容Hive、标准数据连接

sparksql也对上下文环境对象进行了封装，core是SparkContext，而sql是SparkSession

# DSL语法

是DataFrane提供一个特定领域的语言，管理结构化数据。

可以在Scala、java、python、R中使用DSL,使用DSL语法风格不必创建临时视图了

基本语法：

val df=spark.read.json("路径")#创建DataFrane
df.printSchema #查看表结果
df. #tab 查看方法 rdd里面的方法也有
df.select("字段").show #直接根据字段查询
df.select($"age"+1).show #将查询的结果age加1 $表示引用的意思 或者在前面加上单引号(‘age+1)
as #其别名
df.filter('age > 20 )  #过滤
df.groupBY("age").count.show #根据年龄分区，查看条数

强类型自定义函数 (当然还有弱类型，但用的相对较少)

 /**
   * 自定义聚合函数类型
   * 1. 继承org.apache.spark.sql.expressions.Aggregator 定义泛型
   *  IN :  输入数据类型 Login
   *  BUF : 中间计算的临时结构  缓冲区数据类型Buff
   *  OUT : 输出数据类型long
   * 2. 重写方法(8)
   */
  case class Buff(var total: Long,var count:Long)
  class MyAvgUDAF extends Aggregator[Long,Buff,Long]{
    // z & zero 一般是初始值 或 零值
    override def zero: Buff = {
      Buff(0L,0L)
    }

    //根据输入的数据更新缓冲区的数据
    override def reduce(buff: Buff, in: Long): Buff = {
      //更新
      buff.total=buff.total+in
      buff.count+=1
      buff//返回
    }

    //合并缓冲区
    override def merge(b1: Buff, b2: Buff): Buff = {
      b1.total=b1.total+b2.total
      b1.count=b1.count+b2.count
      b1
    }

    //计算结果
    override def finish(buff: Buff): Long = {
      buff.total/buff.count//求平均值
    }

    //todo 基本写法 自定义的类 product 提供的类scalaLong
    //缓冲区的编码操作
    override def bufferEncoder: Encoder[Buff] = Encoders.product
    //输出的编码操作
    override def outputEncoder: Encoder[Long] = Encoders.scalaLong
  }

#使用

    上下文对象.sql("select argeAvg(age) from user").show()

# 数据的加载和保存

spark.read 加载 df.write.save 保存

load #默认读取的文件格式是 只能parquet结尾 
format("json").load("路径")#格式化读取json文件
write.save("路径") #保存数据  parquet文件
write.format("json").save("路径")  #保存json文件

//直接进行读取sql操作
上下文对象.sql("select * from json.’json路径‘ ").show() #也可以操作表
//保存模式 默认情况下是文件不能存在否则抛出异常

#保存

# 连接外部HIVE

  • Spark 要接管 Hive 需要把 hive-site.xml 拷贝到 conf/目录下
  • 把 Mysql 的驱动 copy 到 jars/目录下
  • 如果访问不到 hdfs，则需要把 core-site.xml 和 hdfs-site.xml 拷贝到 conf/目录下
  • 重启 spark-shell

Spark SQL CLI： bin/spark-sql 可在本地运行 Hive 元数据服务，直接执行 SQL 语句

Spark beeline：

基于 HiveServer2 实现的一个 Thrift 服务,无缝兼容HiveServer2

可以和 Hive Metastore进行交互，获取到 hive 的元数据

连接步骤：

  • Spark 要接管 Hive 需要把 hive-site.xml 拷贝到 conf/目录下

  • 把 Mysql 的驱动 copy 到 jars/目录下

  • 如果访问不到 hdfs，则需要把 core-site.xml 和 hdfs-site.xml 拷贝到 conf/目录下

  • 启动 Thrift Server

    sbin/start-thriftserver.sh
    1

  • 使用 beeline 连接 Thrift Server

    bin/beeline -u jdbc:hive2://hadoop111:10000 -n root
    1
