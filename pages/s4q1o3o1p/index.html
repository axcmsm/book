# sqoop

sql+to+hadoop

将关系型数据库与hdfs相互导入导出

官网https://sqoop.apache.org/ (opens new window)

# 原理

将导入导出命令翻译成mr程序运行，主要的是对inputformat和outputformat进行定制

# 安装

  • 下载安装包上传

  • 解压到指定目录

  • 修改配置文件 sqoop-env.sh

    export HADOOP_COMMON_HOME=/opt/module/hadoop-3.1.4
export HADOOP_MAPRED_HOME=/opt/module/hadoop-3.1.4
export HIVE_HOME=/opt/module/hive312
export ZOOKEEPER_HOME=/opt/module/zookeeper-3.6.3
export ZOOCFGDIR=/opt/module/zookeeper-3.6.3/conf
export HBASE_HOME=/opt/module/hbase

# 使用案例

# 导入数据

import 关键字

Rdbms到hdfs：(参数位置可以调换顺序)

  • 全部导入

      bin/sqoop import \
--connect jdbc:mysql://hadoop111:3306/company \
--username root \
--password 密码 \
--table 表名 \
--target-dir 导入hdfs路径 \
--delete-target-dir \
--num-mappers 1 \
--fields-terminated-by "\t"

# target-dir 判断目录是否存在没有就创建 
# delete-target-dir 有就删除
# num-mappers  mapper的个数
# fields-terminated-by 文件分隔符
# 需要指定表名

RDBMS 到 Hive：

 bin/sqoop import \
--connect jdbc:mysql://hadoop111:3306/company \
--username root \
--password 密码 \
--table 表名 \
--num-mappers 1 \
--hive-import \
--fields-terminated-by "\t" \
--hive-overwrite \
--hive-table (hive表名)

# 分成俩部分,一开始先导入hdfs中,然后在迁移到hive中
# 临时目录：/user/用户名/表名
# hive表名没有会自动创建

RDBMS 到 Hbase：

 bin/sqoop import \
--connect jdbc:mysql://hadoop102:3306/company \
--username root \
--password 密码 \
--table 表名 \
--columns "id,name,字段" \
--column-family "info列族" \
--hbase-create-table \
--hbase-row-key "id列名" \
--hbase-table "hbase_company表名" \
--num-mappers 1 \
# --split-by id

# 导出数据

从hadoop集群导出: export 关键字

HIVE/HDFS 到 RDBMS:

bin/sqoop export \
--connect jdbc:mysql://hadoop111:3306/company \
--username root \
--password 密码 \
--table 表名 \
--num-mappers 1 \
--export-dir /user/hive/warehouse/staff_hive \
--input-fields-terminated-by "\t"

#export-dir 数据的路径
#input-fields-terminated-by 分隔符

# 增量

导入

参数：

append：主键

--check-colunm(col) :指定列 例id(主键)
--incremental(mode) :指定追加模式 append
--last-value(value) :id的值后往下的导入

--last-value(value) :指定时间戳值（大于等于)
--append :追加模式 #产生新文件
# --merge-key : 指定字段例如id   #合并成一个新文件
